


```bash
nmap -sn 10.10.10.*
nmap -sn 10.10.10.*  --packet trace # 追蹤
nmap -sn 10.10.10.*  --reason
nmap 10.10.10.9
nmap 10.10.10.9 -p-
nmap 10.10.10.16 -sU -p53,137-139,161,1900,5353
nmap 10.10.10.16 -sU -p53,137-139,161,1900,5353 --open

nmap -sn 10.10.10.*  --reason
nmap ip80 10.10.10.10-12 --reason
nmap -p80 10.10.10.* --open

nmap 10.10.10.16 -O
nmap 10.10.10.16 -sV
nmap 10.10.10.9 -sV # 版本
nmap 10.10.10.16 -sVC -p445,3389 #用腳本掃

nmap -sU -p161 --open 10.10.10.*
nmap -sU -p161 -sC 10.10.10.16
nmap -sU -p161 --script snmp-win32-users 10.10.10.16 # 該主機的帳密

```


```
-sn： No port scan，只看主機有無存在
-sU： udp掃描
-p443：只掃描 443 埠。
-p-65535：掃描所有埠（從 1 到 65535）。
-T0：極慢模式，降低對目標的影響（適用於隱密掃描）。
-T5：最快掃描模式，適合對時間要求高的掃描。
掃描類型
-sA：ACK 掃描，確認防火牆規則或目標是否存活。
-sX：Xmas 掃描，用於探測防火牆規則或檢測埠狀態。
-sT：TCP 連接掃描，完成完整的三次握手。
-sF：FIN 掃描，嘗試繞過防火牆的簡單檢測。
-sP：Ping 掃描，用於快速發現存活主機。
服務與版本探測
-sV：檢測埠上的服務版本。
-sS：SYN 掃描，快速且隱密。
-Pn：跳過 Ping，直接對主機進行掃描。
-V：顯示 NMAP 的版本資訊。
防火牆與隱密掃描
-g：設置源端埠（偽裝為可信任埠）。
-A：啟用高級掃描（包含 OS 偵測、版本探測、腳本掃描等）。
-f：啟用分段掃描，繞過某些防火牆檢測。
-D：使用誘餌主機進行掃描，混淆來源。
作業系統與存活檢測
-O：進行作業系統偵測。
-Pn：不執行 Ping，假設主機存活。
-PU：UDP Ping 掃描，用於檢測主機存活。
-PP：使用 ICMP Ping 掃描。
-PY：SCTP Ping 掃描，檢測是否存活。
解析與輸出格式
-n：不進行 DNS 解析。
-R：強制進行 DNS 解析。
-T：調整掃描速度（0~5，數值越高速度越快）。
-oN：以標準格式輸出掃描結果。
-oX：以 XML 格式輸出掃描結果。
-oG：以易讀的清單格式輸出結果。
其他
--host-timeout 99：設置每個目標的掃描超時時間為 99 秒。
```
